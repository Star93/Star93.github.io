Objective-C

Objective-C 基础知识

总结一些基础的知识。

类方法

Objective-C 中的类方法分为实例方法和静态方法

Objective-C 的方法主要声明在@interface 中，就可以认为是公有的。实际上，Objective-C 没有像 Java ，C++ 中的那种绝对的私有及保护成员方法，仅仅可以对调用者隐藏某些方法。

声明和实现都写在 @implementation 里的方法，类的外部是无法访问的。

Property

Apple 推荐在 Objective-C 中使用 @property 来实现成员变量。
@property 的作用是定义属性， 声明 getter 和 setter 方法。

注意：属性不是变量

@property 可以出现在类的 @interface 块种方法声明的任何地方。还可以出现在 protocol 或者 category 声明中。

上述代码，编译器会帮我们做几件事：

  • 创建实例变量 _name ；
  • 声明 name 属性的 getter 和 setter 方法。
  • 实现 name 属性的 getter 和 setter 方法，同时进行自动内存管理。 Xcode4.5 之前需要手动写 @synthesize 实现属性的 getter 和 setter 方法。之后编译器引入 property autosynthesis ，即属性自动合成。系统会自动实现属性的 getter 和 setter 方法。

@property 的本质： @property = ivar + getter + setter
getter 方法返回值类型必须符合属性类型并且 getter 方法中不能有任何参数。
setter 方法必须带有一个与属性的类型一样的参数，并且返回值的类型为 void 。

@synthesize

表示把变量和属性绑定起来，生成 getter 和 setter 方法并作用于该变量。
@synthesize propertyName = _propertyName

Student.m 中这样写会报错 Use of undeclared identifier '_name' 。

这是因为同时重写了 getter 和 setter 方法，系统就不会帮我们生成这个成员变量了。如果必须要同时重写 getter 和 setter 方法，就需要把未识别的变量用 @synthesize 重新定义，把属性的 getter 和 setter 方法作用于这个变量。
编译时首先会在你的代码中去找，如果手动重写了属性的 getter 和 setter 方法，就用重写的方法，如果没有重写编译器会根据 @property 的额外属性指定的规则去生成相应的 getter 和 setter 方法。

除此之外还有几种需要手动实现 @synthesize 的情况

  • readonly 属性重写了 getter 方法。
  • protocol 中定义的属性。例如 UIApplicationDelegate 中的 UIWindow 。
  • 重载父类中的属性时，也要手动实现 synthesize 。

@dynamic

@dynamic 表示属性的 getter 和 setter方法由用户自己实现（readonly 只需要实现 getter 方法），不需要系统自动生成。如果一个属性被声明为
@dynamic var ，然而没有手动实现 getter 和 setter 方法，编译时不会有问题，但当程序执行到 instance.var = sameVar 或者 someVar = var 的时候就会崩溃。编译时没有问题，运行时才执行相应的方法就是动态绑定。

@property 的额外属性

  • readwrite 可读可写。生成 getter 和 setter 方法。
  • readonly 只读。生成 getter 方法。
  • atomic 原子操作。表示 getter 和 setter 方法执行的时候是一个原子操作，多线程环境下同一时刻只有一个线程可以访问属性。相当于方法头尾加了锁，保证了某种程度上的线程安全。但这种操作耗费系统资源
  • nonatomic 非原子操作 访问速度快。但如果有两个线程同时访问这个属性会出现不可预估的结果。
  • assign 赋值。setter 方法将传入的参数直接赋值给实例变量。
  • retain 持有。setter 方法将传入的参数先保留再赋值，传入参数的 retainCount 会加 1 。
  • copy 拷贝。setter 方法将传入的参数复制一份；传入参数的 retainCount 不会加 1 。
  • strong 持有。类似于 retain。
  • weak 弱引用。传入参数的 retainCount 不会加 1 ，并且在引用对象被释放的时候自动被设置成 nil 。用于 Objective-C 对象
  • getter=getterName 指定属性的 getter 方法名。
  • setter=setterName 指定属性的 setter 方法名。

浅拷贝和深拷贝

浅拷贝：副本和原对象指向同一个地址。原对象的引用计数加 1 。
深拷贝：副本指向的新的地址。原对象的引用计数不变。副本的引用计数加 1 。

原对象数据类型 拷贝方法 副本数据类型 是否产生新对象 拷贝类型
NSString copy NSString NO 浅拷贝（指针拷贝）
NSString mutableCopy NSMutableString YES 深拷贝（内容拷贝）
NSMutableString copy NSString YES 深拷贝（内容拷贝）
NSMutableString mutableCopy NSMutableString YES 深拷贝（内容拷贝）
NSArray copy NSArray NO 浅拷贝（指针拷贝）
NSArray mutableCopy NSMutableArray YES 深拷贝（内容拷贝）
NSMutableArray copy NSArray YES 深拷贝（内容拷贝）
NSMutableArray mutableCopy NSMutableArray YES 深拷贝（内容拷贝）
NS* copy NS* NO 浅拷贝（指针拷贝）
NS* mutableCopy NSMutable* YES 深拷贝（内容拷贝）
NSMutable* copy NS* YES 深拷贝（内容拷贝）

协议 - Protocol

Objective-C 中的继承是单继承，即一个子类只能继承于一个父类。想要实现类似多重继承的效果，可以实现多个 Protocol 协议。任何类只要遵循了 Protocol 协议就相当于拥有了该 Protocol 协议 的所有方法声明。
Protocol 定义了一个方法列表，这个方法列表中的方法可以使用 @required ，@optional 标注，以表示该方法是否是客户类必须要实现的方法。Protocol 本身没有实现。
如果父类实现了某个 Protocol 协议，那其子类可以完全继承其实现。一个 Protocol 协议可以继承其他的 Protocol 协议。
Protocol 协议中可以 用 @property 声明属性。但是属性只有对应的 getter 和 setter 方法声明，并不会生成对应的成员变量。

Protocol 协议的作用

  1. 定义一套公用的接口。
  2. Delegate 传值。Delegate 是 Cocoa 中常见的一种设计模式。

分类 - Category

Category 是一种很灵活的扩展原有类的机制，使用 Category 不需要访问原有类的代码，也无需继承。
Category 提供了一种简单的方式来实现类的相关方法的模块化，把不同的类方法分配到不同的类文件中。这样可以保证类的原始设计规模较小，功能增加时再逐步扩展。

使用注意：

  1. Category 可以访问原始类的实例变量，但不能添加变量。如果想添加变量，可以考虑通过继承创建子类。
  2. Category 可以但不推荐重载原始类的方法。重载后再也不能访问原来的方法。需要重载可以选择创建子类。
  3. Category 可以不必实现所有声明的方法。只要不去调用，不会报错。
  4. 如果有多个命名 Category 均实现了同一个方法，即出现命名冲突，那么这些方法在运行时只有一个会被调用，具体哪个被调用是不确定的。因此在给已有类添加 Category 时，最好加上前缀。

Category 给类添加属性
添加 public 属性来增强 category 的功能。
Runtime 的 associated object 特性。associated object 对象关联是 Objective-C 2.0 运行时的一个特性。
<objc/runtime.h> 中定义的以下三个允许将任何键值在运行时关联到对象上的函数：

相关参数
key ：要保证全局唯一，key 与关联的对象是一一对应关系。必须全局唯一。通常用 @selector(methodName) 作为 key 。
value ：要关联的对象。
policy ：关联策略。是设定 value 在 object 内的额外属性。有五种关联策略。

  • OBJC_ASSOCIATION_ASSIGN 等价于 @property(assign) 。
  • OBJC_ASSOCIATION_RETAIN_NONATOMIC 等价于 @property(strong, nonatomic) 。
  • OBJC_ASSOCIATION_RETAIN 等价于 @property(strong, atomic) 。
  • OBJC_ASSOCIATION_COPY_NONATOMIC 等价于 @property(copy, nonatomic) 。
  • OBJC_ASSOCIATION_COPY 等价于 @property(copy, atomic) 。

扩展 - Extension

Extension 可以认为是一种匿名的 Category 。
Extension 可以声明方法、属性和成员变量。一半用于声明私有方法、私有属性和私有成员变量。
Extension 无法直接为系统类扩展。因为使用 Extension 必须有原有类的源码。可以先创建系统类的子类再添加 Extension 。
