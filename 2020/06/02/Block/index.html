Objective-C

理解 block

block 是 Objective-C 中对闭包的实现。什么是闭包：

In programming languages, a closure is a function or reference to a function together with a referencing environment—a table storing a reference to each of the non-local variables (also called free variables or upvalues) of that function.

翻译：闭包是一个函数（或指向函数的指针），再加上该函数执行的外部的上下文变量（有时候也称作自由变量）。
block 即代码块，将统一逻辑的代码放在一个块中，是一段封装好的代码。
使用 block 可以使代码更简洁紧凑，易于阅读， 比函数使用更加方便，代码更美观。

从三个方面研究 block 的实现原理

  1. block 的内部实现数据结构
  2. block 的三种类型及其相关的内存管理方式
  3. block 如何通过捕获变量达到访问函数外部的变量

数据结构定义

block 实例由6个部分构成

  1. isa - 指针；指向实例对象，表明 block 本身也是一个 Objective-C 对象。
  2. flags - 按位承载 block 的附加信息；
  3. reserved - 保留变量；
  4. invoke - 函数指针；指向具体的 block 实现的函数调用地址。
  5. descriptor - block 的附加描述信息；主要是 size 大小，copy和dispose函数的指针。
  6. variables - 捕获来的变量；block 之所以能够访问外部的局部变量是因为将这些变量（变量的地址）复制到了结构体中。

Objective-C 中的三种 block 类型

  1. _NSConcreteGlobalBlock 全局静态 block；（写在全局作用域的时候，不获取任何外部变量的时候）
  2. _NSConcreteStackBlock 保存在栈中的 block；（当函数返回时会被销毁）
  3. _NSConcreteMallocBlock 保存在堆中的 block；（当 retainCount = 0 时会被销毁）

_NSConcreteGlobalBlock 类型的 block 的实现

_NSConcreteGlobalBlock 类型的 block 在内存的 ROData 段，此处没有局部变量的骚扰，运行不依赖上下文，内存管理也简单的多。

_NSConcreteMallocBlock 类型的 block 的实现

为了解决变量作用域结束时，依然可以继续使用 block 的问题，block 提供了 copy 的功能，将 block 和 __block 修饰的变量从栈拷贝到堆。就是 __NSConcreteMallocBlock 类型的 block 。
__NSConcreteMallocBlock 类型的 block 通常不会在源码中直接出现。

在已有结构体指针指向 __Block_byref_intValue_0 时，结构体里面多了个 __forwarding 指向自己的指针变量。访问结构体 __Block_byref_intValue_0 内部的成员变量都是通过访问 __forwarding 指针完成的。为了保证能正确访问栈上的 __block 变量，进行 copy 操作时，会将栈上的 __forwarding 指针指向了堆上的 block 结构体实例。
