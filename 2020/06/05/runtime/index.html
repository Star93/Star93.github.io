Objective-C

理解 Objective-C 的 Runtime

Objective-C 是一种面向对象的动态语言。动态语言是在运行时来执行静态语言的编译连接的工作。这要求除了编译器之外还要有一种运行时系统来执行编译的代码。这个运行时系统就像一个操作系统一样：他让所有的工作可以正常的运行。Objective-C 中的这个系统就是 Objc Runtime 。 Objc Runtime 其实是一个 Runtime 库，它基本上是用 C 和汇编写的。这个库使得 C 语言又了面向对象的能力。

Runtime 库主要做几件事：

  1. 封装。在这个库中，对象可以用 C 语言中的结构体表示，而方法可以用 C 函数来实现，另外再加上了一些额外的特性。这些结构体和函数被 runtime 函数封装后，我们就可以在程序运行时创建，检查，修改类、对象和方法了。
  2. 找出方法的最终执行代码。当程序执行 [object doSomething] 时会向消息接受者 object 发送一条消息 doSomething ，runtime 会根据消息接受者是否能响应该消息而做出不同的反应。

Class

查看 objc/runtime.h 中 objc_class 结构体的定义如下：

  1. isa ：需要注意的是在 Objective-C 中，所有的类自身也是一个对象，这个对象的 Class 里面也有一个 isa 指针指向 metaClass（元类）。
  2. super_class ：指向该类的父类，如果该类已经是最顶层的根类（如 NSObject 或 NSProxy），则 super_class 为 NULL。
  3. cache ：用于缓存最新使用的方法。一个接受者对象接收到一个消息时，会根据 isa 指针去查找能够响应这个消息的对象。在实际使用中，这个对象只有一部分方法是常用的，很多方法其实很少或者根本用不上。这种情况下，如果每次来消息时，都在 methodLists 中遍历一遍，性能势必很差。所以每次掉用过一个方法后，这个方法就会被缓存到 cache 列表中，下次调用的时候 runtime 会优先去 cache 中查找。如果 cache 中没有才回去 methidLists 中查找这个方法。这样对经常调用的方法提高了效率。
  4. version ：提供类的版本信息。这对于对象的序列化非常有用。它可以让我们识别出不同类定义版本中实例变量布局的改变。

针对 cache ，举个 🌰 来说明其执行过程：

  1. [NSArray alloc] 先被执行。因为 NSArray 没有 +alloc 方法，于是去父类 NSObjcet 查找。
  2. 检测 NSObject 是否响应 +alloc 方法，发现响应，于是检测 NSArray 类并根据其所需要的内存空间大小开始分配内存空间，然后把 isa 指针指向 NSArray 类。同时 +alloc 也被放进 cache 列表中。
  3. 继续执行 -init 方法，如果 NSArray 相应该方法，则直接将其加入 cache 列表；如果不响应则去父类查找。
  4. 之后如果再以 [[NSArray alloc] init] 的方式来创建数组就会直接从 cache 列表中取出相应的方法直接调用。

objc_object 与 id

objc_object 是表示一个类的实例对象结构体。查看 objc/runtime.h 中 objc_object 结构体的定义如下：

这个结构体只有一个字段，即指向其类的 isa 指针。当我们向一个 Objective-C 对象发送消息时。Runtime 库会根据实例对象的 isa 指针找到这个实例对象所属的类。Runtime 库会在类的方法列表及父类的方法列表中去寻找与消息对应的 selector 指向的方法。找到后即运行这个方法。

当创建一个特定类的实例对象时，分配的内存包含一个 objc_object 数据结构，然后是类的实例对象的数据。NSObject 类的 alloc 和 allocWithZone: 方法使用函数 class_createInstance 来创建 objc_object 数据结构。另外还有我们常见的 id ，它是一个 objc_object 结构类型的指针。它的存在可以让我们实现类似于 C++ 中泛型的一些操作。该类型的对象可以转换成为任何一种对象。类似于 C 语言中 void *指针类型的作用。

objc_cache

这是一个指向 objc_cache 结构体的指针。定义如下：

  1. mask ：一个整数，指定分配的缓存 bucket 的总数。在方法查找过程中，Objective-C runtime 使用这个字段来确定开始线性查找数组的索引位置。指向方法 selector 的指针与该字段做一个 AND 位操作 (index = (mask & selector)) 。这可以作为一个简单的 hash 散列算法。
  2. occupied ：一个整数，指定实际占用的缓存 bucket 的总数。
  3. buckets ：指向 Method 数据结构指针的数组。这个数组可能包含不超过 mask+1 个元素。需要注意的是，指针可能是 NULL ，表示这个缓存 bucket 没有被占用，另外被占用的 bucket 可能不是连续的。这个数组可能会随着时间而增长。
