面试题

1. 什么是设计模式？
设计模式是一套被反复使用、多数人知晓、经过分类编目的代码设计经验的总结。简单理解为写代码的套路和模版。（创建型 结构型 行为型）。

2. 项目中使用过哪些设计模式？
工厂方法：工厂父类负责定义创建产品对象的公共接口，而工厂子类则负责生成具体的产品对象，即通过不同的工厂子类来创建不同的产品对象。简化类的初始化过程。
单例：确保一个类只有一个实例变量，并提供一个全局访问点。[NSNotificationCenter defaultCenter]、[NSUserDefaults standardUserDefaults] 等。优点 延迟加载、节省内存。注意使用 @synchronized、dispatch_once 保证线程安全。
代理：为某个对象提供一个代理对象并由这个代理对象控制对原对象的访问。降低代码耦合度。Objective-C 中通常使用 protocol 实现。
观察者：定义对象间的一种一对多的依赖关系，使得每当一个对象的状态发生改变时，其相关依赖对象都可以得到通知并做相应的针对处理。KVO、Notification 。列表滚动偏移量变化监听、网络变化监听。

3. 比较 MVC 、MVP 、MVVM
MVC（Model-View-Controller）：
优点：代码总量少；简单易懂。大部分的代码逻辑放在 C ，M 和 V 严格区分。代码分配遵循一定的规则。
缺点：业务逻辑和视图逻辑混在一起。代码过于集中、测试难度大、可扩展性弱、网络请求层无处安放。
MVP（Model-View-Presenter）：
P 负责业务逻辑。是 M 和 V 的桥接。
优点：耦合度降低、代码分配合理、测试方便。
缺点：V 的所有交互都要传给 P 处理。一旦功能增加，V 和 P 的代码量会翻倍增加，维护成本大。
MVVM（Model-View-ViewModel）：
VM 持有 M 。完全独立于视图 V，V 持有 VM，通过绑定属性自动进行更新。全部操作由响应式逻辑框架自动完成。
优点：分离了业务逻辑和视图逻辑。代码耦合度低、代码量合理。
缺点：需要引入第三方响应式框架，因为属性观察环环相扣调用栈很大，DEBUG 尤为痛苦。

4. #import #include 和 @class 有什么区别？
#import 是 Objective-C 导入头文件的关键字。不会重复导入。#import<> 引入的是系统库的文件；#import"" 引入的是本地工程文件。
#include 是 C/C++ 导入头文件的关键字。
@class 类的声明，解决头文件的相互包含问题。

5. Objective-C 的类是否可以多重继承？
Objective-C 是单继承。可以用协议来代替多继承。可以实现多个接口。

6. Protocol 是什么？
一系列不属于任何类的方法列表。这些方法可以被任何类实现。通过 Protocol 定义各种行为，在不同的场景采用不同的实现方式。一些类之间没有继承关系但又具备某些相同的行为就可以使用协议。

7. Category 是什么？
Category 是一种灵活的扩展原有类的机制。使用分类无需访问原有类的代码，也不需要继承。Category 提供了一种简单的方式来实现类的相关方法的模块化。

8. Extension 是什么？
类的扩展、延展。也叫匿名 Category 。

9. Category 和 Extension 的区别？
使用扩展必须有原有类的源码。可以为系统类添加分类。
使用扩展可以添加属性和实例变量。理论上分类不能添加属性但可以通过 runtime 关联对象实现。
类别有名字、扩展没有。
扩展在编译时期决议，分类在运行时期决议。

10. @property 的本质是什么？ ivar 、getter 、setter 是如何生成并添加到类中的？
@property = ivar + getter + setter
@property 封装对象中的数据。
ivar 、getter 、setter 是自动合成的。
使用 @property 创建一个属性时，编译器会自动编写访问这些属性所需要的方法这个过程就是自动合成。

11. @property 中有哪些属性关键字以及作用？
atomic 一定程度上保证线程安全。
nonatomic 不考虑多线程的情况下，会让编译器少生成一些互斥加锁代码，可以提高效率。
readwrite 可读写。生成 getter setter 。
readonly 只读。生成 getter 。
assign 直接赋值。一般用于基本数据类型。
retain mrc 持有特性 引用计数加 1 。
strong arc 同 retain 。
weak 弱引用。类似于 assign 但是 weak 修饰的对象会在销毁时被置为 nil 。
copy 拷贝特性。setter 方法将之前的对象 relese 再将传入的对象复制一份。
getter= setter= 自定义 getter setter 方法的方法名。

12. 什么情况使用 weak 关键字，相比 assign 有什么不同？
weak 只能用来修饰 Objective-C 对象；assign 一般用来修饰基本数据类型。assign 修饰的对象被释放后，指针的地址依然存在造成野指针。weak 修饰的属性定义了一个非拥有者关系。当给属性设定一个新值时，这个值的引用计数不会加 1 ，旧值的引用计数也不会减 1 。当属性指向的对象被销毁时，该属性会被置为 nil 。

13. copy 和 strong 有什么不同？
strong 的赋值是多个指针指向同一个地址。
copy 是每次会在内存中复制一份对象，指针指向不同的地址。

14. 为什么要用 copy 修饰有对应可变类型的不可变对象而不用 strong ？
因为有对应的可变类型所以两者之间可能进行赋值操作，即把可变的对象赋值给不可变的对象，为了确保对象的值不会跟着赋值对象变动就要在赋值时使用 copy ，让两者指向不同的内存地址。
copy 目的是为了让本对象不受外界的影响。

15. 深拷贝和浅拷贝的区别？
浅拷贝：副本和原对象指向同一个地址。原对象的引用计数加 1 。
深拷贝：副本指向的新的地址。原对象的引用计数不变。副本的引用计数加 1 。

16. Objective-C 的内存管理
内存管理是指软件运行时对计算机内存资源的分配和使用的技术。目的是高效、快速的分配，并且在适当的时候释放和回收内存资源。
引用计数是一种简单有效的管理对象生命周期的方式。
手动引用计数 通过引用计数的方式判断对象是否需要释放。需要开发者手动添加内存管理代码。
自动引用计数 编译源码时编译器会分析源码中每个对象的生命周期，基于分析结果添加相应的引用计数操作代码。
自动内存释放池 把需要释放的对象统一放在一个释放池子中，当池子作用域结束池子中的所有的内存空间也被自动释放掉。

17. 什么是循环引用？
多个对象相互之间强引用，使得引用形成一个环形，导致系统无法释放多个变量。
经常发生在 delegate 、block 、NSTimer 的使用中。
delegate 使用 weak assign修饰
block self 持有的 block ，在 block 中访问的 self 使用 __weak 关键字修饰。
NSTimer [timer invalidate]; timer = nil;

18. self. 和 self-> 有什么区别？
self. 其实是调用了存取方法 setter 或 getter 。
self-> 是直接访问成员变量。
通过存取方法访问比直接访问多做了一些其他的事情（例如内存管理，复制值等）。

19. runtime 有哪些实际作用？
将某些 Objective-C 代码转换为运行时代码。探究底层。
拦截系统的方法替换成自己的实现。例如 viewDidAppear: 、 imageNamed: 等系统方法。
关联对象。实现分类添加属性。
实现 NSCoding 的自动归档解档。
万能控制器跳转。

20. 怎样理解面向对象？
面向过程是站在执行者的角度思考这件事怎么一步一步去完成；
面向对象是站在组织者的角度，它考虑是让谁去做这件事比较合适。以对象为核心，以消息为驱动去实现一件事情。
面向对象的三大特点: 继承，多态，封装。
继承：它可以使用现有类的所有功能，并在无需重新编写原来的类的情况下对这些功能进行扩展。
多态：允许将子类类型的指针赋值给父类类型的指针。不同对象以自己的方式响应相同的消息的能力叫做多态。
封装：对客观事物进行抽象化，隐藏内部实现细节，提供外部接口访问。

20. App 的导航模式有哪些？
平铺模式：一般由 scrollView 和 pageControl 组合而成的展示方式。手机自带的天气比较典型。
标签模式：tabBar 的展示方式，这个比较常见。
树状模式：tableView 的多态展示方式，常见的 9 宫格、系统自带的邮箱等展现方式。

21. UIViewController 的完整生命周期
initWithCoder 通过 nib 文件初始化时触发。
awakeFromNib nib 文件被加载的时候会发生一个 awakeFromNib 的消息到 nib 文件中的每个对象。
loadView 开始加载视图控制器自带的 view 。
viewDidLoad 视图控制器的 view 被加载完成。
viewwillAppear 视图控制器将要显示在 window 上。
updateViewConstrains 视图控制器的 view 开始更新 AutoLayout 约束。
viewWillLayoutSubviews 视图控制器的 view 将要更新内容视图的位置。
viewDidLayoutSubviews 视图控制器的 view 已经更新视图的位置。
viewDidAppear 视图控制器已经展示在 window 上。
viewWillDisappear 视图控制器的 view 将要从 window 上消失。
viewDidDisappear 视图控制器的 view 已经从 window 上消失。

22. 区分 nil 、Nil、NULL、NSNUll
nil：把一个对象置空。空对象。完全从内存中释放。
Nil：nil 表示置空一个对象；Nil 表示置空一个类。
NULL：表示一个空指针。
NSNULL：想把一个对象置空但是又想要一个容器。

23. 区分 define 定义的宏和 const 定义的常量
define 预处理阶段进行内容替换。程序运行时常量表中并没有用 define 定义的宏，系统不会为其分配内存。编译时不会检查数据类型，出错概率大。
const 程序运行时存放在常量表中，系统会为其分配内存，而且在编译时会进行数据类型检查。

24. 区分 UDID 与 UUID
UDID（Unique Device Identifier） 用户设备唯一编码。UDID 是一串由 40 位 16 进制数组成的字符串，用以标识唯一的设备。
UUID（Universally Unique IDentifier） 通用唯一识别符。UUID 是一个 32 位的 16 进制序列，使用小横线来连接：8-4-4-4-12 。它是一种应用加设备绑定产生的标识符。当在设备上安装来自同一个供应商的不同 App 时，此值保持不变。如果你删除了来自某个供应商的所有 App，再重新安装时，此值会改变。获取 UUID 的方法：
NSString *uuidString = [[[UIDevice currentDevice] identifierForVendor] UUIDString];

25. 区分静态库与动态库
库分为开源库和闭源库
开源库：源码公开。方便使用者学习并且根据自己的需求随意修改代码。
闭源库：源码不公开。代码被编译成二进制文件，在编译的时候 link 。根据 link 方式不同，分为静态库和动态库。
静态库：以 .a 、.framework 为后缀。静态库在链接时，会被完整的复制到可执行文件中。多次使用，就会有多次的拷贝。
动态库：以 .dylib 、.framework 为后缀。动态库在链接时不会复制，系统只加载一次动态库，当程序运行时动态加载到内存。多个程序可以共用，节省内存。

KVC 和 KVO
键值编码 Key-Value-Coding 使用字符串来标识属性，间接访问对象的属性，而不是通过调用存取方法直接访问或通过实例变量访问的机制，非对象类型的变量将被自动封装或者解封成对象，很多情况下会简化程序代码。
缺点：一旦使用 KVC 你的编译器无法检查出错误，即不会对设置的键、键路径进行错误检查，且执行效率要低于合成存取器方法和自定的 setter 和 getter 方法。因为使用 KVC 必须先解析字符串，然后在设置或者访问对象的实例变量。
