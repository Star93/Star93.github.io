面试题

1. 什么是设计模式？
设计模式是一套被反复使用、多数人知晓、经过分类编目的代码设计经验的总结。简单理解为写代码的套路和模版。（创建型 结构型 行为型）。

2. 项目中使用过哪些设计模式？

  • 工厂方法：工厂父类负责定义创建产品对象的公共接口，而工厂子类则负责生成具体的产品对象，即通过不同的工厂子类来创建不同的产品对象。简化类的初始化过程。
  • 单例：确保一个类只有一个实例变量，并提供一个全局访问点。[NSNotificationCenter defaultCenter]、[NSUserDefaults standardUserDefaults] 等。优点 延迟加载、节省内存。注意使用 @synchronized、dispatch_once 保证线程安全。
  • 代理：为某个对象提供一个代理对象并由这个代理对象控制对原对象的访问。降低代码耦合度。Objective-C 中通常使用 protocol 实现。
  • 观察者：定义对象间的一种一对多的依赖关系，使得每当一个对象的状态发生改变时，其相关依赖对象都可以得到通知并做相应的针对处理。KVO、Notification 。列表滚动偏移量变化监听、网络变化监听。

3. 比较 MVC 、MVP 、MVVM

  • MVC（Model-View-Controller）：
    优点：代码总量少；简单易懂。大部分的代码逻辑放在 C ，M 和 V 严格区分。代码分配遵循一定的规则。
    缺点：业务逻辑和视图逻辑混在一起。代码过于集中、测试难度大、可扩展性弱、网络请求层无处安放。
  • MVP（Model-View-Presenter）：
    P 负责业务逻辑。是 M 和 V 的桥接。
    优点：耦合度降低、代码分配合理、测试方便。
    缺点：V 的所有交互都要传给 P 处理。一旦功能增加，V 和 P 的代码量会翻倍增加，维护成本大。
  • MVVM（Model-View-ViewModel）：
    VM 持有 M 。完全独立于视图 V，V 持有 VM，通过绑定属性自动进行更新。全部操作由响应式逻辑框架自动完成。
    优点：分离了业务逻辑和视图逻辑。代码耦合度低、代码量合理。
    缺点：需要引入第三方响应式框架，因为属性观察环环相扣调用栈很大，DEBUG 尤为痛苦。

面向对象的设计模式

设计模式（Design Pattern）是一套被反复使用的、多数人知晓的、经过分类的代码设计经验的总结。使用设计模式是为了可重用代码，让代码看起来更容易被人理解，保证代码的可靠性。

设计模式其实就是针对某些问题的一些解决方案。在软件开发中即使有很多人在使用不用的语言去开发不同的业务，但是很多时候遇到的问题抽象出来都是相似的。一些卓越的开发者将一些常出现的问题和解决方案汇总起来，总结出了这些设计模式。

理解 Objective-C 的 Runtime

Objective-C 是一种面向对象的动态语言。动态语言是在运行时来执行静态语言的编译连接的工作。这要求除了编译器之外还要有一种运行时系统来执行编译的代码。这个运行时系统就像一个操作系统一样：他让所有的工作可以正常的运行。Objective-C 中的这个系统就是 Objc Runtime 。 Objc Runtime 其实是一个 Runtime 库，它基本上是用 C 和汇编写的。这个库使得 C 语言又了面向对象的能力。

理解 weak 底层实现

weak 关键字的常见场景是在 delegate ，block 和 NSTimer 中使用，以避免循环引用带来的内存泄漏。但 weak 关键字的方便之处绝不局限于避免循环引用。
weak 的用处用一句话可以归纳为：弱引用。在对象释放后置为 nil ，来避免错误的内存访问。通俗地说，weak 可以在不增加对象的引用计数的同时又使得指针的访问是安全的。

理解 iOS 的内存管理

内存管理是指软件运行时对计算机内存资源的分配和使用的技术。目的是高效、快速的分配，并且在适当的时候释放和回收内存资源。
在 iOS 5 之前，内存管理是令人恐惧的主题，需要花大力气去理解并规划这个部分。需要细致处理引用计数，使用 retain、release 和 autorelease 让应用能够明智的处理内存使用，并在大多数时候不会崩溃。
随着 Apple 引入自动引用计数（Automatic Reference Counting, ARC）特性，我们不再需要花大量的时间去处理引用计数，系统帮我们解决了引用计数的大部分问题。ARC 背后的原理是依赖编译器的静态分析能力，通过在编译时找出合理的位置插入引用计数代码，从而彻底解放程序员。虽然 ARC 帮我们解决了绝大部分引用计数的问题，但还是需要理解引用计数这种内存管理的方式，只有这样才能处理好内存管理的相关问题。

理解 block

block 是 Objective-C 中对闭包的实现。什么是闭包：

In programming languages, a closure is a function or reference to a function together with a referencing environment—a table storing a reference to each of the non-local variables (also called free variables or upvalues) of that function.

翻译：闭包是一个函数（或指向函数的指针），再加上该函数执行的外部的上下文变量（有时候也称作自由变量）。
block 即代码块，将统一逻辑的代码放在一个块中，是一段封装好的代码。
使用 block 可以使代码更简洁紧凑，易于阅读， 比函数使用更加方便，代码更美观。
